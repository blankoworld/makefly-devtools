Makefly

English version (version originale en anglais)

Site web officiel

Description

Makefly est un moteur de blog statique rapide et léger en ligne de commande qui s'appuie sur le format Markdown pour la rédaction et le rendu de ses articles.

Il s'articule autour d'une page d'accueil, d'une liste d'articles, d'une liste de mots-clés et d'une page d'à propos.

Il propose les fonctionnalités suivantes :

  • thèmes
  • flux RSS
  • référencement pour chaque page
  • tags
  • page permanentes/statiques
  • antidatage (poster un article à une date différente)
  • traductions
  • système de commentaire
  • copie de sauvegarde
  • script de publication à distance
  • création rapide de nouveaux thèmes
  • migration depuis Nanoblogger (Cf. section Migration depuis Nanoblogger)

Note : Makefly n'est pas destiné à créer, éditer ou supprimer des billets, même s'il propose un script pour faire un nouveau billet, il est conseillé de se tourner vers des projets tierces tels qu'Hymby (en développement).

Installation

En rapide

En quelques étapes voici comment installer la dernière version (en développement) : 

sudo apt-get install lua5.1 lua-filesystem
curl https://github.com/blankoworld/makefly/archive/master.zip
unzip master.zip
cd makefly-master
cp makefly.rc.fr.example makefly.rc
./makefly clean && ./makefly compile

Vous devriez avoir une liste d'actions effectuées sur votre machine. Et le résultat se trouve dans le dossier pub.

En détaillé

Si vous avez réussi l'étape En rapide avec succès, passez directement à la section Utilisation.

Dépendences

Makefly dépend des élément suivants :

  • lua 5.1 ou plus récent
  • lua-filesystem

Ainsi utilisez le gestionnaire de paquet de votre distribution pour les installer. Par exemple sur Debian et dérivées, ce serait : 

apt-get install lua5.1 lua-filesystem

Pour d'autres distributions, regardez du côté des forums, d'IRC et/ou de la communauté de votre distribution. Ils seront heureux de vous aider.

Autre méthode pour les dépendances

Si votre distribution ne propose pas le paquet lua-filesystem, vous pouvez tenter d'installer luarocks puis installer les dépendances de la manière suivante : 

sudo apt-get install luarocks
luarocks install lua-filesystem

Et le tour est joué !

Installation

Vous avez le choix entre :

  • la version en cours de développement
  • la version stable 0.3 FR (recommandée)

Il suffit donc de :

  • récupérer la version choisie
  • extraire le contenu dans un dossier de votre ordinateur

Et Makefly est installé.

Configuration

Pour la version stable

Aucune configuration particulière n'est requise. Vérifiez que le fichier makefly.rc existe bien. Le cas échéant copiez le fichier makefly.rc.fr.example ou renommez le en makefly.rc.

Pour la version en cours de développement

La première fois que vous utilisez Makefly vous n'avez aucun fichier de configuration. Un exemple de fichier de configuration est disponible dans makefly.rc.fr.example. Copiez le vers makefly.rc pour permettre à Makefly de fonctionner.

Plus d'infos

Pour plus d'informations, lisez la section Le fichier de configuration makefly.rc.

Utilisation

Par défaut le contenu du blog se trouve dans le répertoire pub.

En rapide

Quelques commandes utiles :

  • ./makefly compile : Génére le blog. Disponible dans le répertoire pub
  • ./makefly help : Affiche les commandes possibles
  • ./makefly add : Créer un article et compléter ses infos principales
  • ./makefly clean : Vide complètement le contenu du répertoire final et vide le cache de Makefly

Créer du contenu

Créer un nouvel article

Utilisez la commande suivante :

./makefly add

et répondez à toutes les questions posées. Cela génèrera les fichiers nécessaires pour Makefly.

Il est à noter que Makefly utilise le format markdown pour ses articles.

N'oubliez pas d'éditer le fichier renseigné par la commande './makefly add'.

Contenu statique (photos, vidéos, fichiers PDF, etc.)

Parfois on souhaite partager des fichiers tels quels, que ce soit des images, des démonstrations, des présentations, etc. Il existe pour cela le dossier static. Tout fichier déposé dans ce dossier sera copié dans le dossier de destination.

Exemple :

  • Nous possédons le fichier static/makefly.svg
  • Après compilation du blog, nous avons un fichier pub/makefly.svg

Cela fonctionne donc pour tout type de fichier contenu dans ce dossier.

Pages statiques

Parfois on aimerait ajouter à son blog quelques pages dites statiques. Par exemple les mentions légales.

Pour cela, créez le dossier pages à la racine de votre projet Makefly. Puis ajoutez des fichiers contenant du texte au format Markdown dont l'extension est .md.

Exemple :

  • Nous créeons le fichier pages/mentions.md
  • Après compilation du blog, nous avons un fichier pub/mentions.html

Ceci vous permet de créer un site web avec des pages statiques.

Note 1 : Le nom de la page résultante sera en minuscule et remplacera les espaces par des espaces soulignés (le caractère *_*).

Note 2 : Aucun lien dynamique ne sera fait vers la page statique. Vous devrez modifier vous-même le template pour ajouter un lien vers la page statique. Cf. la section Structure du template.

Le dossier 'special'

Ce dossier nommé special peut contenir certains fichiers que vous devrez créer pour activer une fonctionnalité :

  • about.md : Contient le contenu d'une page d'à propos au sujet de votre site. Cela va ajouter une entrée dans le menu principal du site (si votre thème le supporte).
  • sidebar.md : Ajoute une barre latérale sur votre site. Le thème doit supporter cette fonction.
  • introduction.md : Affiche le contenu de ce fichier comme introduction sur l'ensemble de vos pages. Varie selon le thème choisi.
  • footer.md : Affiche le contenu de ce fichier comme d'un pied de page sur l'ensemble de vos pages. Varie selon le thème choisi.

Générer le blog

Après avoir crée quelques articles, faites simplement : 

./makefly compile

Ceci génèrera un blog Makefly dans le dossier pub (répertoire par défaut).

Publier le résultat sur le web

Le résultat de Makefly est compatible avec tous le serveurs HTML. En effet vous pouvez probablement utiliser le résultat sur le site web de votre hébergeur. Il suffit d'envoyer le contenu du répertoire pub dans celui de votre hébergeur.

Sur un serveur web

Si vous lancez Makefly sur votre propre serveur ou très certainement sur le serveur de votre hébergeur, vous pourrez utiliser l'installation automatisée. Lancez simplement la commande suivante : 

./makefly install

...et cela copiera tous les fichiers dans le répertoire ~/public_html.

ATTENTION: Cela supprimera tous les fichiers contenus dans le dossier public_html !

Note: Vous pouvez personnaliser la destination en changeant le fichier makefly.rc et plus particulièrement la ligne suivante : 

INSTALLDIR=${HOME}/public_html

Relancez ensuite la commande ./makefly refresh && ./makefly install pour recompiler le blog.

Vers un ordinateur distant : la commande publish

Pour publier votre blog vers une machine distante, vous devez :

  • avoir un accès SSH à la machine distante
  • avoir le programme rsync
  • configurer la variable PUBLISH_DESTINATION dans le fichier makefly.rc
  • lancer la commande publish

C'est tout!

À noter que la variable PUBLISH_DESTINATION ressemble à : 

monUtilisateurDistant@domaineDistant.tld:/mondossierhome/dossier_public

Une fois cette variable renseignée dans le fichier makefly.rc, lancez simplement : 

./makefly publish

Pour les dévelopeurs : Vous pouvez aussi modifier le fichier tools/publish.sh et changer le contenu du script par votre propre code.

Créer un nouveau thème

Afin de vous faciliter la tâche de création d'un nouveau thème, vous pouvez utiliser la commande suivante :

./makefly theme myTheme

où myTheme est à remplacer par le nom de votre thème.

Note : Ceci utilise le thème nommé Base comme exemple.

Pour les concepteur de thèmes jetez un œil ici: Documentation sur les thèmes.

Traduction

Une façon simple de traduire Makefly dans votre langage est de copier le fichier lang/translate.en dans un autre fichier. Par exemple, pour le Français (avec le code fr), vous pouvez copier lang/translate.en en lang/translate.fr et changez les valeurs. Puis changez simplement l'option BLOG_LANG dans le fichier makefly.rc.

Sauvegardes

Peut-être voudriez-vous sauvegarder les fichiers importants de Makefly ? C'est possible via la commande backup. Lancez la simplement de cette manière : 

./makefly backup

Requis :

  • tar
  • gzip

Fichiers sauvés :

  • makefly.rc
  • le répertoire static
  • le répertoire special
  • le répertoire db
  • le répertoire src
  • le répertoire contenant le thème choisi (par exemple templates/default/)

Résultat : Ceci créera une archive nommée YYYYmmdd-HM_makefly.tar.gz (20120823-1735_makefly.tar.gz par exemple) dans le dossier mbackup. Vous pouvez ainsi sauvegarder votre Makefly chaque jour.

Astuce

Vous pouvez personnaliser (dans votre fichier makefly.rc) :

  • le dossier de sauvegarde en utilisant l'option BACKUPDIR
  • le préfixe du fichier de sauvegarde en utilisant l'option BACKUP_PREFIX
  • le suffixe du fichier de sauvegarde en utilisant l'option BACKUP_SUFFIX
  • le format de date en utilisant l'option BACKUP_FORMAT

Jouer avec la ligne de commande

Sachant que Makefly fonctionne de la manière suivante : 

./makefly help

et utilise quelques fichiers de configuration comme config et makefly.rc, on peut s'amuser avec quelques variables.

Par exemple : 

LANG=en_US.UTF-8 ./makefly help

Ceci permet de changer la langue de sortie en Anglais.

Ainsi vous disposez des variables suivantes :

  • CURDIR : chemin du répertoire de Makefly. Permet de lancer la compilation depuis un autre répertoire. Par défaut le répertoire courant.
  • LANGDIR : chemin du répertoire contenant les traductions. Par défaut le dossier lang du répertoire courant.
  • LANG : langue utillisée par la ligne de commande. Par exemple en_US.UTF-8 ou fr_FR.UTF-8.
  • CONFIG : chemin du fichier de configuration par défaut de Makefly. Par défaut le fichier config du répertoire courant.
  • RC_CONFIG : chemin du fichier de configuration de l'utilisateur. Par défaut le fichier makefly.rc du répertoire courant.

L'ensemble de ces paramètres vous permettront de créer des scripts utilisant Makefly.

Astuces

Écrire des billets en avance

Dans Makefly vous pouvez écrire des billets en avance. Il suffit pour cela que le fichier de méta-données de votre billet possède un timestamp supérieur à celui du moment où est généré le blog.

Par exemple nous sommes le 6 mars 2013, à 12:30, le timestamp est : 1362569400. Il faut que dans le dossier db, votre article ait un timestamp inférieur à celui d'aujourd'hui.

Écrire directement le contenu de l'article à sa création

Utilisez juste la variable 'content' au début de la commande : 

content="mon petit contenu" ./makefly add

Ceci ajoutera "mon petit contenu" dans votre nouvel article.

Disposer d'un service de commentaires

Makefly dispose d'un service gratuit en bêta test nommé Rave Comment mis à disposition de ses utilisateurs.

Une fois enregistré, vous recevrez une adresse URL telle que : rave.depotoi.re/monpseudo.

Il suffit ensuite d'éditer le fichier makefly.rc et d'y modifier les variables suivantes : 

ISSO = 1
ISSO_URL = rave.depotoi.re/monpseudo

Vous aurez ainsi un système de commentaire sur votre site.

Note : les commentaires ne seront pas visibles sur votre machine. Seulement sur le site distant.

[périmé] Ne pas perdre les commentaires quand on migre d'un vieux domaine à un nouveau

Note : cette astuce n'est plus valable pour la version 0.4 et suivante de Makefly. Elle ne fonctionne que si vous utilisez JSKOMMENT.

Quand vous migrez de vieux.domaine.tld à nouveau.domaine.tld, les commentaires n'apparaîtront plus.

Pour régler le problème, utilisez simplement la commande migratefrom : 

./makefly migratefrom http://vieux.domaine.tld

Ceci va mettre à jour tout les anciens articles avec l'identifiant des vieux commentaires (votre vieux domaine) et les commentaires réapparaîtront.

Le fichier de configuration makefly.rc

Voici quelques options que vous pouvez changer :

  • BLOG_TITLE : Titre de votre blog

  • BLOG_SHORT_DESC : Une courte description de votre blog

  • BLOG_DESCRIPTION : Une description plus complète de votre blog

  • BLOG_LANG : votre code langue. À noter qu'un fichier lang/translate.VOTRE_CODE_LANGAGE doit exister. Par exemple si je configure ce paramètre à fr, un fichier lang/translate.fr doit exister !

  • BLOG_CHARSET : votre configuration d'encodage. Doit ressembler à quelque chose comme UTF-8 ou ISO-8859-1. Si vous ne savez pas ce que c'est, laissez la paramétrée à UTF-8.

  • BLOG_URL : adresse URL absolue de votre blog. Par exemple http://makefly.depotoi.re/.

  • BLOG_AUTHOR : Auteur principal du blog. Permet un référencement dans les moteurs de recherche.

  • BLOG_COPYRIGHT : Copyright du blog. Permet un référencement dans les moteurs de recherche.

  • BLOG_KEYWORDS : Mots-clés qui doivent apparaîtrent pour l'ensemble des pages du blog. Permet un référencement dans les moteurs de recherche.

  • RSS_FEED_NAME : Titre affiché dans le flux RSS.

  • MAX_POST : Nombre maximum d'articles qui seront affichés sur la page d'accueil.

  • MAX_POST_LINES : Nombre de lignes qui seront montrées sur la page d'accueil. Si paramétré à 0 ou inexistant dans le fichier makefly.rc, alors les articles sont entièrement montrés.

  • DATE_FORMAT : Format de la date affichée pour chaque article. Lisez les pages du manuel date pour plus d'informations.

  • MAX_PAGE : Nombre maximal d'articles qui devrait s'afficher sur la liste des articles. Si paramétré à 0 ou inexistant du fichier makefly.rc, alors une seule page sera faite avec tout les articles !

  • SHORT_DATE_FORMAT : Format court de la date. Sera utilisé sur la page de la liste des articles. Pour plus d'informations, lire les pages du manuel date.

  • INDEX_FILENAME : Nom donné à toutes les pages index. Par exemple avec INDEX_FILENAME = mainpage, la liste des articles se nommera mainpage.html.

  • PAGE_EXT : suffixe que toutes les pages auront. NE PAS OUBLIER D'AJOUTER UN POINT AVANT LE SUFFIXE. Par exemple avec PAGE_EXT = .html, toutes les pages seront de la forme : index.html.

  • ABOUT_FILENAME : Comme son nom l'indique, c'est le titre du fichier utilisé pour la page "À propos". Si vous le paramétrez à "apropos" par exemple, vous devez créer un fichier "apropos.md" dans le répertoire nommé special afin de permettre l'obtention d'une page d'à propos. Si vous le changez à toto, vous devez créer un fichier toto.md dans le dossier special.

  • POSTDIR_NAME : Le nom que vous voudriez afficher dans l'URL quand un utilisateur se rend sur la page de la liste des articles. Par exemple, paramétré à "mesarticles" : http://makefly.depotoi.re/mesarticles/ affichera la liste de vos articles. Ceci est utile pour divers langages.

  • TAGDIR_NAME : Même chose que pour le paramètre POSTDIR_NAME, mais pour les mots-clés (tags) cette fois. Modifiez le en "motcle" par exemple et l'adresse suivante affichera la liste des mots-clés : http://makefly.depotoi.re/motcle/.

  • THEME : Nom du thème choisi. Les thèmes sont disponibles dans le dossier nommé template. Chaque thème possède son propre répertoire. Par exemple, le thème default possède son propre répertoire template/default.

  • FLAVOR: Ce nom sera utilisé pour choisir la couleur de votre thème (si elle existe)

  • BACKUPDIR : Nom du dossier où seront sauvés les fichiers résultant de la commande backup.

  • BACKUP_FORMAT : Format de date utilisé pour le fichier de sauvegarde.

  • BACKUP_PREFIX : préfixe utilisé pour le fichier de sauvegarde (entre la date et le nom de fichier).

  • BACKUP_SUFFIX : suffixe utilisé pour le fichier de sauvegarde (entre le nom de fichier et l'extension).

  • SIDEBAR_FILENAME : Comme décrit, nom du fichier utilisé pour la barre latérale. Elle contient des liens et tout un tas d'autres choses. Si vous la paramétrez à "sidebar.md", par exemple, vous devez créer le fichier dans le répertoire special pour obtenir cette barre. À noter que votre thème doit inclure les barres latérales !

  • SIDEBAR : Mis à 1 permet d'activer la barre latérale sur Makefly. À noter que votre thème doit inclure les barres latérales !

  • PUBLISH_DESTINATION : Adresse complète du lieu où envoyer les fichiers afin de les publier.

  • PUBLISH_SCRIPT_NAME : Nom du fichier script utilisé pour envoyer les fichiers du dossier pub vers une destination renseignée dans la variable PUBLISH_DESTINATION.

  • SEARCH_BAR : Mis à 1 permet d'activer une barre de recherche sur Makefly. À noter que votre thème doit supporter la barre de recherche.

  • MAX_RSS : Nombre d'articles RSS maximum qui sera récupéré par vos utilisateurs.

  • ISSO : Mis à 1 permet d'activer un système de commentaires pour Makefly. À noter que votre thème doit supporter le système de commentaires. Attention, par défaut cela utilise rave.depotoi.re en tant que serveur, il ne garantit pas un archivage à long terme des commentaires. Plus d'informations sont disponibles sur la page d'installation du projet isso.

  • ISSO_URL (optionnel) : Définit un serveur ISSO sur lequel envoyer les commentaires. Par exemple rave.depotoi.re/pseudo. Attention : n'utilisez pas de http:// devant l'adresse. Cela permet d'avoir du http ou du https quand nécessaire.

  • ISSO_MAX (optionnel) : Définit une limite de commentaires à afficher pour le système de commentaire ISSO. Par défaut 3.

  • ELI_USER: Si utilisé, ceci active un cadre pour identica. À noter que votre thème doit supporter le widget ELI. Par défaut cette fonctionnalité utiliser l'API d'IDENTICA.

  • ELI_TYPE (optionel) : Changer cet élément par "group" pour suivre un groupe plutôt qu'un utilisateur d'IDENTICA. Par défaut "user".

  • ELI_MAX (optionnel) : Permet de choisir le nombre de statuts affichés. Sur IDENTICA ceci ne peut dépasser 20 éléments. Valeur par défaut : 5.

  • ELI_API (optionnel) : Accès à l'API de votre système StatusNet.

  • INSTALLDIR : Permet de choisir le dossier de destination lors de l'utilisation de la commande ./makefly install (Cf. Chapitre Publier le résultat sur le web).

  • SORT (optionnel) : Tri la liste des billets. Utilisez ASC pour que les billets soient triés du plus anciens au plus récent. DESC (valeur par défaut) tri les billets du plus récent au plus ancien.

  • AUTO_EDIT (optionnel) : Permet d'éditer automatiquement les billets après leur création. Utilise le contenu de la variable EDITOR pour savoir quel éditeur utiliser.

Le projet

Description

Makefly est un sous-projet de BlogBox qui vise à fournir de meilleurs moyens pour héberger un blog à la maison.

Site web

Visitez fréquemment le blog de Makefly (en) pour avoir des nouvelles du projet.

Alerte rouge, bug détecté !

Vous avez trouvé un bug ? Ou quelque chose ne va pas (hormis vos problèmes de couple) ? C'est parti pour ouvrir un rapport de bug sur Github. C'est simple :

  • Allez sur ce lien
  • Si vous n'êtes pas enregistré, utilisez le lien Sign up. Puis recliquez sur le lien précédent pour accomplir les étapes
  • Cliquez sur New Issue
  • Donnez un titre qui serait une courte description de votre problème
  • Puis, sous le titre, expliquez en détail :
    • Ce que vous avez fait
    • Le résultat que vous obtenez
    • Ce à quoi vous vous attendriez plutôt
    • [optionnel] donnez un copier/coller de ce que vous avez à l'écran ou donnez une capture d'écran
  • Validez en utilisant Submit new issue

This take few minutes and permit to improve Makefly. Thanks -in advance - a lot for your help!

Astuce pour forker/bifurquer le projet

Le projet peut se dupliquer et se renommer facilement en utilisant les variables suivantes dans le fichier makefly :

  • PROJECTNAME
  • PROJECTURL
  • GITPROJECT

Et en renommant le fichier makefly par le nom donné dans PROJECTNAME en minuscule.

Pensez également à renommer le fichier makefly.svg en celui du nom de votre projet.
